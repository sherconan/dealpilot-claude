# DealPilot 产品方法论

> 把全球 VC 共识的 BP 筛选方法论落到一个可操作的工作台 — 这一份是产品的"宪法"

---

## 一、底层信念

VC 筛选的核心赌注始终是「人」 — 但人不可量化，所以行业沉淀出三层逐步逼近的框架：

1. **Sequoia 10 要素清单**（事实标准）：使命 / 问题 / 解决方案 / 时机 / 市场 / 竞争 / 商业模式 / 团队 / 财务 / 愿景
2. **a16z 技术 × 文化双维度**：技术洞见 + 创始人 × 市场契合度 + 时机 + 网络效应
3. **YC 极简三件事**：技术能力 + 市场理解 + 创始人特质

DealPilot 的评分模型在三套框架之上做加权融合。详见 `src/lib/scoring.ts`。

## 二、100 分 Scorecard 设计

总分 100，分四大类、12 子项：

### 赛道吸引力（20%）
| 子项 | 权重 | 评分逻辑 |
|---|---|---|
| 市场规模 TAM | 8% | ≥ $10B = 10；$1-10B = 7；< $1B = 3 |
| 市场增速 CAGR | 7% | ≥ 20% = 10；10-20% = 7；< 10% = 3 |
| 政策支持度 | 5% | 国家战略鼓励 = 10；中性 = 7；受限 = 0 |

### 团队质量（30%）
| 子项 | 权重 | 评分逻辑 |
|---|---|---|
| 创始人经验深度 | 10% | Top 10 公司 ≥ 5 年 = 10；≥ 3 年 = 7；< 3 年 = 3 |
| 团队完整性 | 8% | 4 职能齐全 = 10；缺 1 = 6；缺 2+ = 0 |
| 历史成功记录 | 7% | 有成功退出 = 10；大公司高管 = 7；首次 = 3 |
| 团队稳定性 | 5% | 联创磨合 ≥ 1 年 = 10；< 1 年 = 5 |

### 商业模式（23%）
| 子项 | 权重 | 评分逻辑 |
|---|---|---|
| 收入模型清晰度 | 8% | 已有付费 + 模式清晰 = 10；假设收入 = 5 |
| 单位经济学 | 7% | LTV/CAC ≥ 3 = 10；1-3 = 5；< 1 = 0 |
| 护城河宽度 | 8% | 专利 / 网络效应 / 独家资源 = 10；品牌 = 7；无壁垒 = 0 |

### 增长动能（15%）
| 子项 | 权重 | 评分逻辑 |
|---|---|---|
| Traction 证据 | 10% | ARR ≥ $1M 且增速 ≥ 100% = 10；有收入低增速 = 5 |
| 客户留存 NRR | 5% | NRR ≥ 120% = 10；≥ 100% = 7；< 100% = 0 |

### 估值与条款（12%）
| 子项 | 权重 | 评分逻辑 |
|---|---|---|
| 估值合理性 | 7% | 同阶段比较合理 = 10；偏高 = 5；严重偏高 = 0 |
| 退出可见度 | 5% | 2 种清晰退出 = 10；1 种 = 7；模糊 = 0 |

### 总分分级
- **80-100**：优先推进（Priority Deal）— 排入 IC + 锁定份额
- **65-79**：持续观察（Monitor）— 季度回访 + 信号监控
- **50-64**：有条件跟进（Conditional）— 改进后再评
- **< 50**：建议 Pass — 机构记忆库存档

## 三、Red Flag 清单

### 硬 Red Flag（命中即 Pass）
- 市场规模数据无来源 / 严重夸大（差 10 倍以上）
- 竞争分析为空白（"我们没有竞争对手"）
- 财务数据前后矛盾（增速 vs 营收规模不对应）
- 创始人 / 高管离职率高 / 创始人已套现离场
- 未披露的诉讼 / 税务 / 知识产权纠纷
- 商业模式依赖单一大客户（占比 > 50%）
- 估值脱离基本面（无营收要 10 亿 +）
- **创始人前公司有合规问题且未在 BP 披露**

### 软 Red Flag（扣分项）
- BP 逻辑不自洽（前后模块互相矛盾）
- 团队无相关行业背景
- 过度依赖政策补贴
- 产品 Demo 无法复现 BP 中的描述
- 创始人无法清晰解释核心技术
- 融资用途不清晰或过于分散

## 四、漏斗模型

```
初筛 (5–15 min)         跟进 (1–2 周)          尽调 (2–4 周)         IC (1 次)            出关 TS (1–2 周)
─────────────────────────────────────────────────────────────────────────────────────────
头部 VC 通过率 ~1%   →  约 10–20%          →  ~50%               →  ~50–60%           →  TS 发放
重点：市场 / 团队 /     重点：合伙人愿意       重点：财务真实性     重点：退出路径 /     核心：估值 / 董事
为什么是现在            Champion + 客户       团队历史 + 商业        条款谈判 / 风险      会席位 / 优先权 /
                       访谈 + Demo            模式验证               缓释                  反稀释
```

每阶段都对应 DealPilot 中的具体 stage：`inbox / review / dd / ic / invested / pass`。

## 五、IC Memo 标准结构

DealPilot 自动生成的 IC Memo 严格遵循 8 段式：

1. **执行摘要** — 头部信息（估值/金额/条款）+ 3 个核心投资论点 + 推荐建议
2. **市场分析** — 动态分析（为何现在）+ 需求信号 + 竞品格局
3. **团队评估** — 5 维度：能力 / 行业经验 / 热情 / 团队互补 / 创业经验
4. **产品 / 牵引力** — 真实客户数据 + 产品差异化
5. **单位经济学** — 收入模型 + 随规模改善逻辑 + 关键风险假设
6. **风险与缓释** — 具体可接受的缓释措施
7. **投资结构与退出路径** — 优先股 / 董事会 / 反稀释 / 退出
8. **附录** — Sequoia 10 要素详细评分

> 核心定位：IC Memo 是**信念文件（Conviction Document）**，帮助合伙人做出更好决策；不是防御性文件，不是为了保护写作者免责。

## 六、AI 筛选边界

| 领域 | AI 擅长 | 人类必须介入 |
|---|---|---|
| 数据处理 | ✅ 快速解析、跨源验证、异常检测 | — |
| 模式识别 | ✅ 基于历史成功 / 失败模式打分 | — |
| 初步筛选 | ✅ 高通量过滤、优先级排序 | — |
| 市场研究 | ✅ 聚合数据、竞品分析 | — |
| 创始人评估 | — | ⚠️ 韧性、远见、领导力、信任感 |
| 关系建立 | — | ⚠️ VC 本质是信任生意 |
| 逆向创新 | — | ⚠️ AI 基于历史训练，天然低估颠覆性 |
| 价值观判断 | — | ⚠️ 道德 / 伦理决策 |

> Alpha HUB 原则：**AI 不替代投资判断 — 它让判断更锐利。** 目标是把 GP 的时间集中在最重要、最有把握的机会上。

## 七、2026 新增门槛

- **AI 原生能力**：产品核心价值由 AI 模型直接产生，非传统软件附加 AI
- **ESG 合规**：从加分项变为基金层面合规要求
- **地缘合规**：涉及中美双向技术敏感的项目需通过审查
- **创始人反脆弱性**：在逆周期能否存活到下一个窗口
